# Key Features for Ecosystem Adoption

To become a standard units interoperability layer in Scientific Python,
PyUnitWizard should eventually support the following capabilities.

1. Backend interoperability across Pint, OpenMM, Astropy, and unyt.

2. Canonical dimensional representation shared across backends.

3. Safe interoperability with NumPy numerical operations.

4. Automatic plotting compatibility with Matplotlib.

5. Canonical serialization format for quantities.

6. Safe extraction of numerical arrays from quantities.

7. Integration utilities for pandas DataFrames.

8. Cross-library dimensional compatibility checks.

9. Conformance test suite for backend integrations.

10. Performance optimizations (lazy conversions, caching).

11. Clear error reporting for dimensional mismatches.

12. Stable interoperability API usable by external scientific libraries.

Together these features would position PyUnitWizard as a core
infrastructure component for handling quantities with units
in the Scientific Python ecosystem.
