# External Best Practices

This appendix links to external guidelines that complement PyUnitWizard's own
developer standards.

## Recommended resources

- [MolSSI Python Package Best Practices](https://education.molssi.org/python-package-best-practices/)
- [MolSSI Best Practices (AIO version)](http://education.molssi.org/python-package-best-practices/aio.html)
- [MolSSI Education portal](https://education.molssi.org/)
- [MolSSI best-practices source repository](https://github.com/MolSSI-Education/python-package-best-practices)

## How this relates to PyUnitWizard

These resources are useful for packaging, CI, and contributor workflows.
Project-specific behavior and release contracts are still defined by this
repository's developer pages and `devguide/` documents.
